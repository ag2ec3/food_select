# "오늘 뭐 먹지?" — 제품·기능 명세

---

## 1. 개요

### 문제 정의

팀 단위로 점심(또는 식사) 메뉴를 정할 때 흔히 발생하는 문제:

- **눈치 싸움** — 먹고 싶은 게 있어도 먼저 말하기 어렵다.
- **우유부단** — "아무거나"를 반복하다 시간만 낭비한다.
- **반복·편중** — 목소리 큰 한 명의 취향으로 매번 같은 메뉴가 선택된다.

### 타깃 사용자

- 매일 점심 메뉴 선택으로 시간을 허비하는 **직장인 팀** (3~20명 규모)
- 서로 눈치를 보느라 의견을 내지 못하는 팀 문화

### 핵심 가치

| 가치 | 설명 |
|------|------|
| **빠름** | 제안 → 투표 → 확정까지 몇 분 안에 완료 |
| **공평** | 1인 1표, 동점 시 랜덤 — 특정인의 의견이 과대 반영되지 않음 |
| **간편** | 복잡한 설정 없이 팀에 들어와서 바로 사용 |

---

## 2. 용어 정의

- **팀(Team)** — 함께 식사하는 그룹 단위. 하나의 팀은 독립된 데이터 공간을 가진다.
- **세션(Session)** — "오늘의 메뉴 결정" 한 회차. 하루에 여러 세션을 열 수도 있다(점심·저녁 등).
- **후보(Candidate)** — 세션 안에서 팀원이 제안한 메뉴 항목.
- **투표(Vote)** — 팀원이 원하는 후보에 찍는 표. 1인 1표.
- **마감** — 투표를 종료하는 행위. 자동 타이머 없이 **"마감" 버튼**으로만 수행한다.
- **확정 메뉴** — 마감 후 최다 득표(동점 시 랜덤)로 결정된 최종 메뉴.
- **이력(History)** — 과거 세션의 확정 메뉴 기록.

---

## 3. 기능 목록

### 3-1. 메뉴 제안

- 세션이 열린 상태에서 팀원 누구나 먹고 싶은 메뉴를 **자유롭게 텍스트로 입력**하여 후보를 추가한다.
- 중복 메뉴명은 허용하지 않는다(같은 세션 내).

### 3-2. 투표

- 후보 목록에서 원하는 메뉴를 선택해 투표한다.
- **1인 1표** — 이미 투표한 경우 다른 후보로 변경할 수 있지만, 복수 선택은 불가하다.

### 3-3. 마감

- 투표 종료는 자동 시각이 아니라 **"마감" 버튼**을 눌러 수동으로 진행한다.
- 마감 권한은 세션을 연 팀원 또는 팀 내 누구나(MVP 기준) 가진다.

### 3-4. 확정 로직

- 마감 시점의 득표를 집계하여 **최다 득표 메뉴를 확정**한다.
- **동점**인 경우 동점 후보 중 **랜덤으로 1개**를 선택한다.

### 3-5. 확정 메뉴 공유

- 확정된 메뉴는 해당 세션 화면에서 **팀 전체가 즉시 확인**할 수 있다.
- 세션 상태가 "확정"으로 전환되며, 확정 메뉴가 강조 표시된다.

### 3-6. 이력 조회

- 팀의 **최근 확정 메뉴 이력**을 목록으로 조회할 수 있다.
- 각 이력에는 날짜, 확정 메뉴명, 득표 수 등 요약 정보를 표시한다.

---

## 4. 비기능 요구사항 / 제약

- **사용 목적** — 식사 메뉴 결정에 특화된 도구로, 범용 투표 도구가 아니다.
- **인증** — Supabase Auth 기반. 팀 가입/초대를 통해 팀에 소속된 사용자만 참여 가능하다.

---

## 5. 예외 처리 / UX

| 상황 | 동작 |
|------|------|
| 후보가 0개인 상태 | 목록 영역에 **"메뉴를 작성해주세요"** 안내 문구를 표시한다. |
| 후보 0개에서 마감 시도 | 마감 버튼을 비활성화하고, **"메뉴를 작성해주세요"** 동일 메시지로 안내한다. |
| 투표 없이 마감 | 후보가 1개 이상이면 마감은 허용한다. 득표 0인 후보끼리 랜덤 확정. |

---

## 6. MVP 범위 vs 이후 확장

### MVP (v1)

- 팀 생성·가입
- 세션 생성·메뉴 제안·투표·마감·확정
- 확정 메뉴 팀 내 공유 (앱 내 화면)
- 이력 조회

### 추후 확장

- 알림 (푸시, 이메일)
- Slack / 카카오톡 등 외부 메신저 연동
- 카테고리·태그 (한식, 중식 등)
- "오늘은 이거 빼줘" 제외 기능
- 통계 (가장 많이 선택된 메뉴, 팀원별 취향 등)

## 7. 데이터 엔티티 초안

> Supabase(PostgreSQL) 기준. 실제 마이그레이션 시 컬럼 타입·인덱스는 조정한다.

| 엔티티 | 주요 필드 | 비고 |
|--------|-----------|------|
| **Team** | `id`, `name`, `created_at` | 팀 기본 정보 |
| **Membership** | `id`, `team_id`, `user_id`, `role`, `joined_at` | 팀-사용자 다대다 관계 |
| **Session** | `id`, `team_id`, `status`(open / closed), `created_at`, `closed_at` | 한 회차의 메뉴 결정 단위 |
| **Candidate** | `id`, `session_id`, `user_id`, `menu_name`, `created_at` | 팀원이 제안한 후보 메뉴 |
| **Vote** | `id`, `session_id`, `user_id`, `candidate_id`, `voted_at` | 1인 1표 (session + user 유니크) |
| **Decision** | `id`, `session_id`, `candidate_id`, `decided_at` | 확정 결과. 이력 조회의 기본 단위 |
